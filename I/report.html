1. Introduction

1.1 Project Scope

The goals of the project is to support developers in collecting, interrelating and finding resources of interest, i.e. specialized websites, tutorials, multimedia presentations, source code, software projects etc., targeting creative programming: creative coding depending on the purpose, the programming languages (e.g., JavaScript, WebGL, WebAssembly), the desired result (2D illustration, 3D virtual world, animation, sound content), the manner of interaction (classical, tactile, vocal) and others.

Based on the purpose ("I want to create landscapes using fractals", "I need a polyphonic generator") and/or the user's preferences (e.g., "only JavaScript/TypeScript implementations"), the system will recommend existing software solutions (frameworks, libraries, components etc.) necessary to create an artifact.

The platform, therefore, bridges the gap between complex programming concepts and tangible creative outputs. By facilitating access to relevant resources based on specific project needs or user preferences, it encourages experimentation, creativity, learning, and, ultimately, the development of unique digital artifacts.

2. Overall Description

2.1 Product Perspective

This platform is a pioneering, standalone product designed to significantly enhance the accessibility and usability of resources for creative programming. It serves as an intermediary between developers and a vast array of educational and practical resources, utilizing cutting-edge natural language processing to offer tailored recommendations. The system's uniqueness lies in its ability to interconnect with both local databases and external APIs like OpenAI, providing a seamless integration for users seeking to advance their programming skills in creative and innovative ways.

2.2 Product Functions

  • Natural Language Query Processing
  • Resource Recommendation
  • OpenAI Profile Integration

2.3 User Classes and Characteristics

Different user classes include Guest Users, who have access to basic functionalities and a limited number of recommendations queries, Authenticated Users, who enjoy personalized recommendations tailored to their needs and preferences with OpenAI integration and Administrators, who oversee the platform's operations.

Each class has, therefore, distinct interaction levels and privileges.

2.4 Operating Environment

The software is designed to operate in a web-based environment, utilizing technologies such as Node.js for the backend and HTML, CSS, and JavaScript for the frontend. It is compatible with various hardware platforms (PC, tablets, mobile devices) and operating systems, offering broad accessibility for its users.

2.5 Assumptions and Dependencies

Key assumptions include the reliability of the OpenAI API for processing queries and accessibility to a MySQL database for the resources and various user preferences. Dependencies are primarily on the OpenAI API for its natural language processing capabilities.

3. External Interface Requirements

3.1 User Interface

The platform will feature a simple, responsive web design, compatible with various devices and screen sizes (no screen layout constraints). The user interface will also include a dark/light theme toggle for user eye-comfort.

Basic sketch of the UI

3.2 Hardware Interfaces

As a web-based application, there are no specific hardware interface requirements. The platform is accessible and performant on standard computing devices including desktops, laptops, tablets, and smartphones.

3.3 Software Interfaces

The platform will interface with the OpenAI API for processing natural language queries. The backend database will be MySQL and will structure the data according to schema.org recommendations, whereas the server-side logic will be implemented in Node.js. (WIP other technologies)

4. System Features and Requirements(WIP)

4.1 Personalized Recommendations

Users can input their project goals into a form, which is processed in natural language by the OpenAI API. The system will compare these inputs with the locally hosted MySQL database and will recommend the relevant resources, frameworks and tools, followed by a description covering key aspects of each.

4.2 Resource Submission and Management

Authenticated users have the ability to submit resources to the platform. Each submission will require the following information: (WIP)

  • Title of the resource
  • Description
  • URL
  • Relevant Tags (e.g., JavaScript, WebGL, 3D Animation)

Submissions will undergo a review process by administrators before being made publicly available.

4.3 User Preferences and Profile Management

Users can manage their profiles, including personal information, preferences, and linked OpenAI API credentials. This section also allows users to review and manage their submitted resources and preferences for recommended technologies.

4.4 Administrator Capabilities

Administrators have access to a dashboard for supervising platform activity. This includes user management, resource approval(WIP), and other system settings. Features include:

  • User account creation, suspension, and deletion
  • Resource review and management
  • System analytics and reporting
